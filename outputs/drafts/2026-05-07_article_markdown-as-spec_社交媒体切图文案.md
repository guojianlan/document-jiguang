# Markdown as Spec 社交媒体切图文案

下面 5 段每段独立成图，适合切成单图发即刻 / X / 朋友圈 / 小红书。每段含一个核心洞察 + 一句反问或反预期，便于截图传播。

---

## 切图 1：开篇钩子（最适合做封面图）

> **4 个独立工具，1 个共同答案。**
>
> OpenSpec、DESIGN.md、AGENTS.md、SKILL.md，4 件没人协调过的事，全走到了同一个格式——markdown 文件，仓库固定路径，AI 启动前注入。
>
> 这种独立收敛通常意味着背后有真实需求结构，不是巧合。

反问钩子：

> 你猜下一个会出现的契约文件叫什么名字？

---

## 切图 2：5 条共性（适合做矩阵图配图）

> **markdown-as-spec 的 5 条共性：**
>
> 1. 都是 markdown，不是 JSON / DSL
> 2. 都靠文件系统交付，不靠 API
> 3. 都把人类规则前置到 AI 启动之前
> 4. 都在跨厂商收敛
> 5. 都是"动作 + 标准"的二维结构

反预期 hook：

> 单独有 skill，agent 不知道按什么标准做。单独有标准，agent 不知道怎么做。两件事拼起来才是完整工作流。

---

## 切图 3：知识沉淀模式的转折（适合发独立开发者社群）

> **以前你写文档是给三个月后的自己看。现在是给明天进来的某个 agent 看。**
>
> 设计师的视觉判断 → DESIGN.md
> 架构师的业务规约 → OpenSpec
> 工程师的协作约定 → AGENTS.md
> Senior 的工作流经验 → SKILL.md
>
> 这是知识沉淀模式的一次转折——直接以"AI 可消费"为目标书写。

反问钩子：

> 你手里那些"我习惯这样做"的私人方法论，是不是也该 markdown 化了？

---

## 切图 4：未来候选清单（适合配预测话题）

> **按这个 pattern 推下去，未来还会出现的几个 *.md 契约——**
>
> - **PROMPT.md**：项目级 prompt 规约
> - **VOICE.md**：品牌语气规范
> - **SECURITY.md**：威胁模型 + 边界
> - **DATA.md**：数据 schema + 隐私边界
>
> 不是预言哪一个一定出现，是这个模板已经成熟，会被复制到其他领域。

反预期 hook：

> 看到下一个新工具，先猜它是不是某种 .md。猜对的概率比你以为的高。

---

## 切图 5：收尾金句（适合做转发卡片）

> **写给 AI 看的 markdown 契约，正在变成跟代码同等重要的工程产物。**
>
> 十年前我们学会写 README，五年前我们学会写 CONTRIBUTING.md，现在轮到 AGENTS.md / SKILL.md / DESIGN.md 这一波。
>
> 工具会迭代，pattern 会留下来。

反问钩子：

> 下次新接 AI 工具，先问一句——它的契约文件叫什么名字、放在哪、长什么样。

---

## 使用建议

- **切图 1** 适合做朋友圈首图 / 公众号封面 / 即刻头条卡
- **切图 2** 配认知图发，给读者一个完整框架
- **切图 3** 发到独立开发者社群（独立黑客 / indie hackers / 即刻创作者）转化最高
- **切图 4** 押预测的 hook，半年后回看可以做"打脸 vs 兑现"复盘
- **切图 5** 适合做"读完再转发"的封尾卡，可同时附文章链接

排版建议：

- 切图正文统一深色背景 + 米色字（与认知图风格保持一致），让 5 张图有"系列感"
- 反问钩子单独一行加大字号，与正文区分
- 每张图右下角附文章标题 + 一个二维码或短链，转化更稳
