# 社交媒体切图文案——Claude Code 内部 12 组件

每条都是可独立传播的金句，建议做成方形（1080×1080）或竖版（1080×1350）切图。

---

## 切图 1（钩子型 · 反直觉数字）

> **12 章源码，0 处 if-else。**
>
> Claude Code 没有任何一行代码替模型做"下一步该干什么"的决策。
>
> 模型决定调什么工具、何时停、要不要 load skill、何时主动 compact。
>
> harness 写的全部是工具、记忆、边界——一个能让模型干活的"世界"。

适用——X / Twitter 首贴、知乎卡片、小红书图文首图。

---

## 切图 2（哲学宣言型）

> **Agency 来自训练，不是来自编排。**
>
> 这是 shareAI-lab 在 learn-claude-code README 里反复论证的一句话。
>
> 12 章源码风格高度一致——没有一个分支节点、没有一个状态判断、没有一个"如果模型说 X 就做 Y"的 if 块。
>
> 信这个的人写薄 harness，不信的人去拖 LangGraph 节点图。

适用——朋友圈、社群转发、知识星球开篇。

---

## 切图 3（架构判断型）

> **循环属于 agent，机制属于 harness。**
>
> Claude Code 的循环不到 30 行 Python，11 章源码里这个循环一行没改。
>
> s02 加工具，循环不动。s03 加 todo，循环不动。s06 加压缩，循环不动。
>
> 内核只有一个，能力靠工具堆。

适用——技术博客封面、开发者群第一张引流图。

---

## 切图 4（设计判断型）

> **会话短命，磁盘长命。**
>
> Claude Code 后半段 7 章（s06-s12）全部在做一件事——把跨会话/跨 agent 的协调落到比对话长命的存储。
>
> transcript、task、mailbox、worktree——全是磁盘文件。
>
> 因为会话上下文随时会被 context compact 抹掉。

适用——X / Twitter 二贴、文章中段切图、长文订阅渠道。

---

## 切图 5（实操判断型 · 可选）

> **抄 Claude Code 时该抄什么。**
>
> 做"专门写 SQL 的 agent" → 抄 s01-s05（循环、工具、todo、subagent、SKILL.md）。
>
> 做"长期常驻、跨会话恢复"的 agent → 上 s06 + s07 + s11（压缩、任务图、自治轮询）。
>
> 单 agent 别碰外层 7 章，硬抄会把项目复杂度抬一倍而 0 收益。

适用——做实际项目的开发者群、agent 工程实战话题下。

---

## 使用建议

- 5 条金句任选 2-3 条做切图，不要全发——会显得在 spam
- 优先发 1 + 4（钩子数字 + 设计判断），转化最稳
- 配图风格建议——黑底白字 / 米色底深色字，跟正文配的认知图色系（米色 + 橙 + 绿）一致
- 每条切图末尾可加一行小字"完整文章在 → [文章链接]"，引流到正文
