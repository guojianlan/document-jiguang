# 来源到下游衔接规则

这个文件定义 `source-auto-summary` 在完成来源笔记后，什么时候应该继续往下游走。

## 目标

避免来源处理永远停留在 `SourceNotes`。  
来源笔记完成后，应判断它是：

- 到此为止
- 该进入主题笔记
- 该进入文章 / 发布包装

## 默认判断规则

### 情况 1：只做来源笔记即可

满足任意一项时，可以先停在来源笔记：

- 用户只要求摘要
- 当前只有单条来源，且主题还不稳定
- 来源本身只是事实快照或临时参考

### 情况 2：应继续进入主题笔记

满足任意两项时，建议继续进入主题层：

- 已经有 2 条及以上来源指向同一问题
- 来源内容已经出现稳定重复观点
- 用户明确在做某个长期主题
- 这条内容明显不只是新闻，而是可持续积累的方法或认知

### 情况 3：应继续进入文章或发布包装

满足任意一项时，建议继续进入发布链路：

- 用户明确说要写文章、做分享、做系列内容
- 已经有稳定主题笔记
- 这条内容本身就是为输出服务的内容生产任务

进入发布链路后，优先交给：

- [article-publish-kit](/Users/apple/Desktop/project/document/skills/article-publish-kit/SKILL.md)

## 默认话术方向

完成来源笔记后，后续 agent 可以按这类逻辑继续：

- “这条来源我已经整理成来源笔记。如果你愿意，我可以继续把它并入主题笔记。”
- “这几条来源已经足够支撑一个主题，我建议继续沉淀主题笔记。”
- “当前内容已经具备成稿条件，下一步更适合交给发布包装流程。”

## 一个原则

不要把所有内容都直接推向文章。  
更稳的顺序永远是：

来源 -> 主题 -> 成稿 -> 发布包装
