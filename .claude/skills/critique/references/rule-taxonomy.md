# 37 模式分类法（中文化适配版）

本文件是 `/critique` skill 的规则骨架。结构借鉴自：
- [Aboudjem/humanizer-skill](https://github.com/Aboudjem/humanizer-skill) (MIT) — 37 模式分类
- [blader/humanizer](https://github.com/blader/humanizer) (MIT) — 重写策略与二轮自审

英文条目已重写为中文等价物。每个 P# 在 `/critique` 评分中归到一个核心维度，命中后按对应维度扣分。

## A. 内容模式 P1-P8（影响**信息密度**与**独特视角**）

### P1 重要性膨胀
> "标志着关键时刻" / "具有里程碑意义" / "见证了……" / "彰显了……价值" / "在……中起着不可忽视的作用"

提炼问题：用宏大词替代具体事实。
改写：去掉重要性宣告，直接给读者能验证的事实。

### P2 名头堆叠（伪权威）
> "曾被多家媒体报道" / "拥有活跃的社交媒体" / "广受业界认可"

提炼问题：用模糊名头制造权威感。
改写：要么给具体的某次报道、某家媒体；要么删掉。

### P3 -ing 化粗浅分析（中文对应：进行式状语堆叠）
> "彰显着……" / "推动着……" / "助力着……" / "强调了……"

改写：变成主谓宾的具体动作。

### P4 推销腔
> 前沿 / 顶尖 / 一流 / 卓越 / 行业领先 / 沉浸式 / 无缝衔接 / 焕然一新

改写：换成可观察的具体特征。

### P5 模糊归因
> "专家认为" / "研究表明" / "业内普遍认为" / "据相关数据显示"

改写：要么给具体的人 / 报告 / 链接；要么删掉。

### P6 模板段（挑战 / 机遇 / 展望）
> "尽管面临挑战，仍持续繁荣" / "未来前景广阔" / "机遇与挑战并存"

改写：清零。读者不需要这种填充段。

### P7 AI 高频词
> 详见 [zh-ai-tells.md](./zh-ai-tells.md) 第 4、5、10 节（分析动词、降维引导语、伪学术腔）

### P8 系动词回避
> "扮演着……角色" / "起着……作用" / "充当着……" / "代表着……"

改写：直接用"是 / 有 / 做"。

## B. 语言风格 P9-P18（影响**阅读节奏**）

### P9 否定平行（"而是"句）
> 详见 [zh-ai-tells.md](./zh-ai-tells.md) 第 1 节，全文 ≤ 1 次。

### P10 强行三段论
> "创新、灵感与洞察" / "速度、深度与广度" / "效率、品质与体验"

改写：拆成自然的搭配，不强凑三个并列。

### P11 同义词循环（指代变换）
> 同一主体用三五个不同名词反复指代（公司 / 这家企业 / 该机构 / 此公司 / 团队）

改写：固定一个名词 + 代词。

### P12 伪范围（X 到 Y）
> "从初创公司到上市企业" / "从入门到精通" / "从小白到大神"

只在 X 与 Y 真处于一个连续谱系上才用，否则改写。

### P13 破折号过载
> 一段里出现 ≥ 3 个破折号，或几乎每个长句都靠破折号续接

改写：多数破折号换成逗号、句号或括号。

### P14 加粗 / emoji 过载
> 每段都有加粗，或每个标题都顶 emoji

改写：保留真正需要强调的 1-2 处加粗，删 emoji。

### P15 列表病
> `**主题：** 描述` 这种把每个要点都包装成"标签 + 解释"的结构 ≥ 5 条

改写：能用流畅段落讲完的就别用列表。

### P16 标题题干化
> 每个 H2/H3 都是完整句子或 Title Case

改写：用短语式标题，不强求每个标题都有动词。

### P17 排印细节
> 中文里使用全角括号 + 大量"——"破折号 + 弯引号 + 强制 Oxford 逗号

改写：跟随仓库默认（直引号 / 半角标点 / 句号代替部分破折号）。

### P18 正式语域过载
> "需要指出的是" / "应当注意" / "有必要强调" / "不可忽视的是"

改写：直接给观点，删掉所有"需要指出"式前缀。

## C. 沟通腔 P19-P21（影响**独特视角**与**阅读节奏**）

### P19 客服式套语
> "希望这能帮到你" / "当然可以" / "好的，没问题"

改写：清零。

### P20 知识截止声明
> "截至我的训练数据" / "根据已有信息" / "在可获取的资料中"

改写：清零。

### P21 谄媚腔
> "这是个非常好的问题" / "你提的角度非常棒"

改写：清零。

---

## D. 注水 / 退让 P22-P30（影响**信息密度**与**阅读节奏**）

### P22 注水短语
> "为了……" → "想……" / "由于……的原因，导致……" → "因为……所以……" / "在当前这个时间点" → "现在"

改写：去掉冗余介词结构。

### P23 过度退让
> "可能潜在地或许" / "也许某种程度上"

改写：留一个程度副词就够。

### P24 通用结尾
> "前景一片光明" / "未来值得期待" / "让我们拭目以待"

改写：换成具体的下一步动作或具体预测。

### P25 幻觉痕迹
> 编造的日期、不存在的论文、查不到的人名

改写：找不到原始来源就删。

### P26 完美 / 错误交替
> 全文行文一致，但局部突然出现明显错字 / 标点错配 / 重复字

改写：通读一遍，统一风格。

### P27 问句标题
> "什么让 X 如此独特？" / "X 究竟意味着什么？"

改写：直接给陈述句标题。

### P28 Markdown 标记泄漏
> 在飞书 / 公众号粘贴版里仍然出现 `**加粗**` 字面字符

改写：导出前确认目标平台是否支持 Markdown。

### P29 "全面综述"开场
> "本文将深入探讨……" / "让我们一起来看……" / "今天和大家分享……"

改写：清零，直接进入第一句观点。

### P30 句长一致
> 通篇句子 15-25 字，节奏均匀

改写：人为打破。塞 1-2 个 5 字以下短句，塞 1 个 40 字以上长句。

## E. 新兴模式 P31-P37（多维度影响）

### P31 主语指代变换
> 同一主体反复换称呼（这个项目 / 该计划 / 这次实践 / 此举）

改写：固定一个称呼。

### P32 协作口吻泄漏
> "我们将一起探讨" / "让我带你了解" / "下面我会逐步说明"

改写：清零。中文文章里作者就是作者，不需要把读者拉进"我们"。

### P33 占位符泄漏
> `[你的名字]` / `[请插入链接]` / `[此处补充数据]`

改写：发布前必须填入或删除。

### P34 引用标记泄漏
> `citeturn0search0` / `oai_citation` / `【1†source】`

改写：发布前清扫。

### P35 URL 参数泄漏
> 链接里残留 `utm_source=chatgpt.com` 或 `?ref=ai`

改写：清理参数。

### P36 风格 / 语域突变
> 前两段冷静分析，第三段突然变鸡汤；或前文严谨，后文开始用网络梗

改写：通读统一语域。

### P37 名头-空内容
> "曾被《某某》报道，并出现在多家媒体" 但没有任何具体内容

改写：要么补具体内容，要么把名头删掉。
