# 来源摘要输出规范

这个文件定义 `source-auto-summary` skill 处理来源时的最小交付标准。

## 共同要求

所有来源笔记都应满足：

- frontmatter 中 `status` 应从 `draft` 更新为 `processed`
- 保留原始来源信息
- 一眼能看出“这份内容到底讲什么”
- 区分事实、观点和你的判断
- 给出是否值得进入主题笔记的建议
- 给出是否适合继续进入发布链路的建议

## 不同来源类型的完成标准

### article

至少补齐：

- 一句话摘要
- 核心观点，至少 3 条
- 关键事实 / 数据 / 论据，至少 3 条
- 方法 / 框架 / 流程
- 我认为最值得保留的内容
- 与哪些主题相关

### link

至少补齐：

- 快速判断
- 摘要
- 关键信息
- 对我的价值
- 关联主题

如果只是一个事实快照，要明确写出这是“事实更新”还是“观点表达”。

### video

至少补齐：

- 一句话摘要
- 结构拆解
- 关键观点，至少 3 条
- 重要案例 / 金句 / 表达
- 可转成什么内容
- 关联主题

### pdf

至少补齐：

- 一句话摘要
- 目录 / 结构
- 关键结论
- 关键数据 / 图表 / 证据
- 可复用框架
- 我的判断
- 关联主题

### document

至少补齐：

- 一句话摘要
- 结构拆解
- 关键信息
- 可复用内容
- 我的判断
- 关联主题

## 状态定义

- `draft`：只完成入库，尚未补全摘要
- `processed`：已完成基础摘要和结构化提炼
- `published`：该来源已进入对外输出或发布内容

## 失败处理

如果来源无法读取或缺少正文，来源笔记里应明确记录：

- 为什么当前无法完整摘要
- 还缺什么输入
- 下一步建议

## 下游建议格式

来源笔记完成后，建议至少补一句“下一步建议”，可选方向包括：

- 继续补同主题来源
- 并入某个主题笔记
- 继续生成文章草稿
- 继续进入发布包装
