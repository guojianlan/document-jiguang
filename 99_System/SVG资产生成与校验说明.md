# SVG 资产生成与校验说明

这个说明用于约束仓库里的封面图、认知图、结构图和图卡类资产。

## 默认原则

- `SVG` 作为源文件，优先保留可编辑性
- 对文本密度高的结构图，优先保留 `数据文件 + 模板` 作为更高优先级的生成源
- 生成 SVG 后，必须先渲染成 PNG 预览检查一次
- 对文本型 SVG，默认再补一次布局检查，优先捕捉明显的文字越界和重叠
- 确认没有文字裁切、换行异常、对齐偏移、字号失衡后，才视为完成
- 如果目标平台需要位图分发，再从已验证的 SVG 导出 PNG

## 为什么不默认只做 PNG

- SVG 更适合后续改字、改配色、改布局
- 同一份源文件可以继续导出不同尺寸
- 你的仓库后续会持续复用模板，保留可编辑源文件更划算

所以默认策略不是“SVG 或 PNG 二选一”，而是：

`SVG 作为源文件 -> PNG 作为预览或分发产物`

如果目标平台是飞书文档、公众号编辑器或其他富文本系统，默认应优先引用 `PNG`，不要直接引用 `SVG`。

## 最小校验项

每次生成 SVG 后，至少检查：

- 标题和英文单词是否被裁切
- 中文是否出现不自然换行
- 卡片标题和正文是否超出容器
- 左右留白是否失衡
- 重点文字是否真的足够醒目

## 本地校验命令

```bash
bash scripts/validate_svg_asset.sh "07_Attachments/xxx.svg"
```

这个命令现在默认会在渲染预览后继续执行一次布局检查：

```bash
node scripts/check_svg_layout.js "07_Attachments/xxx.svg"
```

如果希望顺手导出一个 PNG：

```bash
bash scripts/validate_svg_asset.sh "07_Attachments/xxx.svg" --png "07_Attachments/xxx.png"
```

如果要按发布宽度导出，而不是按 SVG 原始宽度导出：

```bash
bash scripts/validate_svg_asset.sh "07_Attachments/xxx.svg" --png "07_Attachments/xxx.png" --width 1440
```

如果只是做校验预览，建议命名为：

`原文件名.preview.png`

当前固定约束改为：

- 最终 PNG 尺寸必须等于 SVG 根节点的 `width` 和 `height`
- 浏览器渲染只负责保证文字和布局观感
- 默认发布 PNG 可以直接按目标宽度导出，并按原始比例自动换算高度
- 不允许让当前机器的窗口高度直接决定最终 PNG 高度
- 不允许依赖“先截大图再手工裁切”作为默认发布方案

这意味着导出结果不应因为当前机器屏幕尺寸或浏览器窗口大小而变化。

推荐宽度约定：

- 横版封面图 / 结构图：`1440`
- 竖版图卡：`1080`

如果图中文字较多，建议优先读取：

- [Satori视觉资产说明.md](/Users/apple/Desktop/project/document/99_System/Satori视觉资产说明.md)

从现在开始，更明确的规则是：

- 文字密度高、容器较多、容易超框的图，首次生成就优先使用 `Satori`
- 手写 SVG 更适合图形为主、文字为辅的资产
- 如果文本型 SVG 已经被布局检查识别出明显风险，优先改模板或数据，不优先手调坐标

但需要注意另一层差异：

- 如果 SVG 使用的是系统字体栈，不同机器安装的字体不同，文字观感仍可能略有差异
- 如果要追求跨机器几乎一致的视觉结果，应优先使用稳定字体方案，必要时把关键标题转路径或嵌入字体资源

## 适用范围

- 文章封面图
- 认知图
- 结构图
- 社交媒体图卡
- 系列内容的视觉母版
