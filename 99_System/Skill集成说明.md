# Skill 集成说明

本仓库现在区分两种 skill：

## 1. 仓库内 skill 草稿

用于设计、孵化和版本管理，位置在：

- `skills/<skill-name>/`

这些 skill 属于项目资产，适合：

- 持续迭代
- 跟随仓库一起维护
- 先在本项目里验证

## 2. Codex 全局 skill

真正提供给 Codex 触发使用的 skill，通常位于：

- `$CODEX_HOME/skills/...`

如果仓库内的 skill 已经成熟，可以再考虑迁移或同步为全局 skill。

## 推荐流程

1. 先在本仓库内设计和验证 skill
2. 确认触发条件、输入输出、资源结构都稳定
3. 再决定是否集成到全局 skill 目录

## 集成前检查

- 是否已经有清晰的 `SKILL.md`
- 是否有必要的脚本或参考资料
- 是否已经经过真实任务验证
- 是否值得长期维护

## 一个务实原则

不要一开始就把所有想法都做成全局 skill。

更好的方式是：

- 先在仓库里孵化
- 真正跑通
- 再决定是否全局化
