# 业务闭环面板说明

这个面板是 `business_loop` 系统的本地可视化入口，用来在一个界面里管理多个业务仓库。

它适合下面这种使用方式：

- 先登记或初始化一个仓库
- 说明这个仓库是做什么的
- 声明允许写入哪些目录
- 声明默认测试命令
- 直接输入自然语言需求或 PRD 路径
- 启动运行并观察状态
- 如果业务规则不清，系统自动阻断
- 在项目记忆里持续维护业务规则

## 启动方式

推荐命令：

```bash
python3 -m business_loop.cli serve-panel --open
```

也可以直接使用脚本：

```bash
bash scripts/start_business_loop_panel.sh
```

或者使用 npm 入口：

```bash
npm run loop:panel
```

默认地址：

```text
http://127.0.0.1:8765
```

## 面板当前能力

- 查看已登记项目
- 初始化新项目
- 登记已有项目
- 查看项目用途、业务域、技术栈、允许写入目录、默认测试命令
- 编辑项目记忆
- 直接输入自然语言需求启动运行
- 直接指定 PRD 路径启动运行
- 查看最近运行记录
- 查看运行状态、阶段明细和阻断信息

## 项目注册表

为了让同一个面板管理多个外部仓库，系统会维护一个全局注册表。

默认位置：

```text
~/.business-loop-control/registry.json
```

如果设置了环境变量 `BUSINESS_LOOP_HOME`，则会优先使用该目录。

## 项目目录内的长期文件

每个被初始化的项目目录下会生成：

- `.business-loop/project.json`
- `.business-loop/memory.md`

其中：

- `project.json` 用来描述这个项目的业务身份和执行边界
- `memory.md` 用来长期维护业务规则、口径、约束和人工审核要求

## 当前适用边界

当前更适合：

- 本地项目
- 你能明确项目根目录的位置
- 你希望人工管理边界、自动执行实现和测试
- 你接受在规则不清时系统自动停止

当前还不是：

- 完整桌面应用
- 带原生文件夹选择器的 GUI
- 跨机器统一调度中心

如果后续需要更像桌面应用，可以继续在这个面板基础上包一层桌面壳。
