# 卡兹克写作指纹（A 结构 + B 语言）

3 篇样本：行业反思 / 工具教程 / 产品评测 各 1 篇，覆盖他三类主体内容。统计指标已剔除文末固定模板段。

## 量化基线

| 维度 | 篇 1 反思 | 篇 2 教程 | 篇 3 评测 | 我们当前两稿 |
|---|---|---|---|---|
| 字数 | 2536 | 3299 | 3131 | 4000+ / 2500 |
| 段数 | 106 | 137 | 115 | ~25 / ~30 |
| **段均字数** | **23.9** | **24.1** | **27.2** | **80-120** |
| 配图 | 20 | 31 | 31 | 1 / 1 |
| "我"次数 | 62 | 47 | 48 | <10 / 8 |
| "你"次数 | 15 | 58 | 36 | 多但非对话 |
| 显式分章节 | 0 | 12 | 0 | 6-7 |

**最大差距**：他段均 24 字 + 30 张图，我们段均 100 字 + 1 张图。一个是"刷下去"的节奏，一个是"读下去"的节奏。

## A 结构层 6 条指纹

### A1. 极短段 + 高密配图

每 80-100 字配 1 张图，每段 1-2 句话就换行。手机屏幕滑下去几乎每屏都换段，**视觉疲劳成本被压到极低**。我们当前是技术博客式长段，公众号场景天然吃亏。

### A2. "昨天，X" 起手锚定时间

3 篇全用——"昨天，GPT-image-2 正式上线了" / "昨天眼看到小伙伴…" / "昨天，Claude 凌晨的时候…"。**强制时间在场感**，让读者觉得"这是新鲜事，不是知识普及"。比"最近 / 这段时间"更狠。

### A3. 内容类型决定是否分节

教程类（10 个隐藏命令）显式 "一./二./.../十./写在最后"；反思和评测类全流式叙事，0 章节。**节奏跟着情绪走，不跟着结构走**。我们的稿子无论什么类型都强分节，会显得"PPT 化"。

### A4. 工位段子 / 同事故事开篇

教程篇不上来讲 `/rewind` 是什么，先讲"同事 prompt 写错了我说你咋不用回退他来了一句啥是回退然后我就懵了"。**钩子是人，不是技术名词**。

### A5. 收尾三段式：长解释 → 短句堆叠 → 口号

固定配方，3 篇都用：

```
[一段稍长的总结判断]
[3-5 个 4-8 字的短句，独立成段]
[最后一句口号 / 金句 / 4 字成语]
```

样例：
- 篇 1："我只能说，在这个黑暗森林里，做一个值得被信任的人。 / 这可能是我们唯一能做的事。 / 也可能是，最重要的事。"
- 篇 3："恐怖如斯。 / 在如今，你想花 20 美刀… / 相信我，至少在今天。 / Claude，YYDS。"

**这是他最强的结构指纹**——读者读到结尾会被节奏带出"小高潮"，比平铺直叙的总结记忆点高一个量级。

### A6. 情绪密度 > 信息密度

3000 字里至少 5-6 处"灌情绪"段（震惊 / 焦虑 / 兴奋 / 自嘲）。**信息密度反而比技术博客低**——他用情绪锚点撑读完率，不靠信息密度。我们当前稿是反过来——信息密度撑独特性，几乎没有情绪锚点。

## B 语言层 8 条指纹

### B1. 第一人称密度 1 次/50 字

平均每 40-50 中文字 1 个"我"。**作者强在场感**，不是分析者腔，是"老哥跟你聊"腔。我们当前稿是"我读完得出 X"——他是"我读完简直惊了，因为 X"。

### B2. 短句独立成段做强调

"猜疑链闭环了。" / "恐怖如斯。" / "疯了吧。" 一句话独立成段。**节奏断点**，强迫读者在这里停一下。我们的稿是"长句中间塞强调"，断点感弱。

### B3. "但，"另起短段做小转折

不写"但是 / 然而"，写"但，"——逗号紧跟，单字成段起头。"我并不是说所有人立刻怀疑每张图。 / 但，那个怀疑的种子，已经种下了。" **比标准转折词更轻、更口语、更强烈**。

### B4. 公理化 / 数学化句式压判断

"当造假的成本趋近于零的时候，信任的成本就趋近于无穷大。这是一个非常基本的数学问题。" 把社会观察压成"公理"句式。**让感性判断带上理性外壳**，是他经常出现的招牌句。

### B5. 物理化动词

"摁在地上打" / "刀刀砍在命脉上" / "把对手甩在身后"。**评测结论必须用身体性动词**。我们的稿是"性能更强 / 优于"——他是"摁在地上打"。

### B6. 口语爆点词高频

"究极 / 真的 / 贼 / YYDS / 狗东西 / 小龙虾 / 整活 / 老子"。每篇 5-10 个。**牺牲一点专业感换强亲和力**。但注意——他选词是"圈层准"，玩梗能让目标读者会心一笑，不是滥用。

### B7. 故意保留口语笔误 / 长省略号 / 波浪号

"我们甚至都都不需要意识到" 的"都都"没改；"。。。。。。" 六个句号；"Coding 愉快～" 的波浪号。**打的是真人手抖痕迹**，不是 AI 写的就能放心继续读。

### B8. 自创 / 圈内黑话给产品起绰号

"小龙虾"指 Claude（Claw-de），整篇贯穿。**专属代号 = 读者门票**——能听懂的人就是自己人。这一条不能硬抄，要养自己的代号。

## 跟我们当前两稿对照

| 维度 | 卡兹克 | markdown-as-spec（我们）| 12 components（我们）|
|---|---|---|---|
| 段均字数 | 24 | ~100 | ~80 |
| 第一人称密度 | 1/50 字 | <1/500 字 | 1/200 字 |
| 收尾节奏 | 长 → 短 → 口号 | 平铺总结 | 平铺心法句 |
| 情绪锚点 | 5-6 处 | ~1 处 | ~1 处 |
| 物理化动词 | 多 | 0 | 0 |
| 口语爆点 | 多 | 0 | 极少 |
| 配图密度 | 1 张/100 字 | 1 张/全文 | 1 张/全文 |
| 风格定位 | 老哥跟你聊 | 同行写笔记 | 同行写笔记 |

**结论**：我们当前两稿是"高质量行业博客"风格，卡兹克是"高情绪公众号长文"风格。**两条路线都成立，但发布场域不同**——博客风格在掘金/少数派/微信深度群好读，公众号场景"刷不动"，他的风格反过来。

## 改写示例

取 12 components 文章原段（line 5）：

> 原文：
> GitHub 上 shareAI-lab 的 `learn-claude-code` 把这件事掰得很清楚——12 节课，12 个递进的机制，每章在前一章基础上叠加一个东西。我读完 12 章源码（每章 100-800 行 Python，加起来 4000 多行），读完反而发现，外界普遍流传的那张"12 组件平铺图"是错的。

按卡兹克风格重写：

> 昨天熬夜读了 4000 行代码。
>
> shareAI-lab 这个 repo `learn-claude-code`，12 节课，每节 100-800 行 Python，把 Claude Code 内部拆得明明白白。
>
> 然后我傻了。
>
> 你在网上能搜到的所有"Claude Code 12 组件图"，全是错的。
>
> 真的，全是。
>
> 那张被疯转的 12 宫格平铺图，从根上就拍歪了。

差异点：
- 段从 1 段变 6 段
- 加 "昨天熬夜" 时间钩子
- "我傻了 / 真的，全是" 短句独立成段
- "拍歪了" 物理化动词
- 第一句具体数字（4000 行）抛在最前

## 怎么用这份指纹

不是写每篇都要"卡兹克化"。**按目标场域选风格**：

| 发布场域 | 风格基线 |
|---|---|
| 公众号 / 小红书 / 朋友圈 | 卡兹克风格至少抄 4-5 条（短段 + 钩子 + 收尾三段 + 第一人称） |
| 掘金 / 少数派 / 知乎专栏 | 我们当前博客风格够用，可以在开篇 + 收尾局部抄卡兹克 |
| 内部 wiki / 技术文档 | 当前博客风格更合适，不要"灌情绪" |

写作 sub-agent 在 publish-article skill 里被调用时，**应根据目标平台决定是否引用本文件**。后续如果加 voice 类 skill，把"按平台选风格"作为第一步判断。

## 反例 / 警惕

- **硬抄会翻车**：他的"小龙虾 / YYDS / 狗东西"是养出来的圈层 token，新作者直接用会显得低龄。先建自己的口头禅再说。
- **段段灌情绪 = AI 味**：他的情绪段是"该灌时狠灌"，不是"每段都激动"。中段大量是平静叙述，靠对比衬托高潮。
- **章节去掉不等于流式叙事就成立**：他的流式叙事每段之间有清晰的"小转折"逻辑。直接删章节但段落之间没转折，会变成流水账。
