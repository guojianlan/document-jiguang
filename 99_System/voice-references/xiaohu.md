**这是小互最显著的特征：双模写作**。两种类型的稿子语言层差异比同一作者其他维度的差异都大。

## A 结构层 6 条指纹

### A1. "深度解读"和"实测复盘"两套模板，分别对应

公众号「小互AI」当前可见两类内容：
- **A 类·深度解读**：给海外创业公司 / 论文 / RFS 做翻译 + 评论。模板：起手讲一个具体场景 → 引出某海外人物 / 公司 / 论文 → 拆 3-5 层架构 → 每层带原文金句 → 结尾"留个问题给你"
- **B 类·实测复盘**：拿自己的 Agent / 工具跑 N 个场景 → 每场景一个小标题 + 数据落地结果 → 结尾"四个实验背后的同一个逻辑"做总结 + 延伸"还能玩什么"

两类共有的母版：**讲完事实立刻给"翻成一句人话"的口语化注解**。

### A2. 数字编号大章节："01 / 02 / .../ 09"

深度解读篇用 9 个独立"01-09"章节硬切。每个章节标题是问句或断言（"公司不是因为缺数据失败的，是因为缺记忆" / "Agent 接的是一个不记得自己的公司"）。**章节标题就是观点本身**，不是中性的"功能介绍 / 案例分析"。

跟卡兹克（教程才分章）和歸藏（轻分章 + 疑问句）都不同——**小互是把章节当"小高潮编号器"用**。

### A3. 实测篇的"实验 1 / 2 / 3 / 4 + 落地结果三件套"

OpenClaw 实测篇结构非常工整：
- 实验 1：双色球（验证"抓→统计→报告"链路）
- 实验 2：选题日报（验证"日常产出"）
- 实验 3：竞品分析（验证"业务场景"）
- 实验 4：门店巡检（验证"线下场景"）

每个实验末尾必带"落地结果"小段，列出：
- 已建表名 + 写入条数
- 字段构成
- 已生成报告
- 仍有问题的部分

**这是产品复盘的标准格式**，比歸藏的 5 案例平铺更结构化（带验收标准）。

### A4. 收尾"留个问题给你" 反向钩子

深度解读篇结尾不是金句，是 3 个连续问题：
- "你公司的「公司大脑」现在长在哪几个老员工的脑子里？"
- "那几个人离职那天，公司会瞬间忘掉多少？"
- "要是张三明天走，要是小李后天走，公司需要多久..."

实测复盘篇结尾也是问题："评论区告诉我，你最想拿它管什么？"

**反向钩子**：不抛结论，抛问题让读者带走。这点跟卡兹克的"长 → 短 → 口号"和歸藏的"链接 + 转发请求"都不同——**小互是"把问题塞回读者脑子里"**。

### A5. 引用密度高 + 来源链接全标

深度解读篇大量引用：
- McKinsey 报告（带年月：2026 年 4 月《Building the Foundations for Agentic AI at Scale》）
- 1991 年 James Walsh 的《Organizational Memory》论文
- YC RFS 条目（带操刀人：Tom Blomfield / Diana Hu）
- Granola / Otter / Glean / Zapier 各家产品策略
- Kahneman 的 dual-process model

每个引用都带具体来源（人名 / 公司名 / 时间 / 论文标题），**给"翻译者口径"做信任背书**。文末必带"原文：https://x.com/...""**让读者能溯源**。

跟卡兹克 / 歸藏几乎不引用（卡兹克只引同行、歸藏只引自己）形成强对比。

### A6. 段落"块状叙事"+ 单句"断点"混合节奏

不像卡兹克段段都短，也不像歸藏段段都列项。小互是：
- **核心论述用 80-150 字的"块"**：3-5 个长句叠在一起，把一个判断讲透
- **关键节点用 1 句独立成段做"断点"**：让读者停下、消化

样例（公司大脑篇）：
```
[长块] 大多数系统把这些当成各自独立的东西存。Slack 是 Slack，CRM 是 CRM，PRD 是 PRD。
       Ashwin 的判断是公司大脑必须保留它们之间的关系。

[断点] 人走了，它就消失了。

[长块] 这事 AI 不来，问题没那么明显。Sarah 和 Ravi 一边干活一边把上下文写一点出来...
```

**这种"块 + 断点"节奏让长文不疲劳**。是他撑得起 7000 字的关键技术。

## B 语言层 8 条指纹

### B1. "翻成一句人话"是招牌动作

深度解读篇出现至少 3 次"翻成大白话"或"说人话就是"：
- "Ashwin 这篇里最值得拎出来的一句，翻成大白话：「Agent 失败不只是因为公司缺数据...」"
- "Ashwin 给这个能力起了个名字叫 metacognition，说人话就是「想自己有没有想清楚」。"
- "翻成一句人话：你不再是手工搬运信息的人..."

**先讲学术 / 原文 / 英文术语，再立刻"翻译"成口语**。这是他给"读不动 paper / 看不懂英文推"的中文读者搭的桥。**翻译者口径**是他最强的差异化标签。

### B2. 大白话压判断（不是术语，是俗语）

短句独立成段时，他的句子是俗语化判断，不是金句：
- "记忆不是档案。"
- "数据可以买，可以爬，可以洗。 / 判断链不能。"
- "信息不再丢了。"
- "记账这件事终于不需要意志力了。"

跟歸藏"约束越严，风格越稳"的审美权威感不同，跟卡兹克"恐怖如斯"的情绪爆点也不同——**小互的短句是"白话锚点"**，让读者能复述给别人。

### B3. 双重否定 + 反问组合

"它不只是忘掉事实，它会忘掉为什么这件事当初重要" / "靠的不是 CRM 里那些字段。靠的是 Sarah 还记得..."

这种"不是 X，而是 Y" 的对照句式高频。**句式本身就在做"纠正普遍认知"的动作**——这跟"深度解读"的定位相符（不是给你新信息，是纠正你已经有的理解）。

注意：这条命中我们仓库 critique skill 里"而是"句的硬规则，**抄的时候要警惕变 AI 味**。小互能撑住，是因为他每次"不是 X 而是 Y"后面都接具体例子；如果只用句式不带例子，会立刻像 AI 写的。

### B4. 第二人称"你公司 / 你的 Agent / 你打开 CRM" 拉近距离

深度解读篇 25 个"你"，大部分是"你公司""你打开 CRM""你的 Agent"——**把读者塑造成"决策者 / CEO / 创始人"**。这条比卡兹克的"老哥跟你聊"格局大，跟歸藏"你可以试试"的工具引导口气也不同。

实测复盘篇里"你"则切换成"你不再是手工搬运信息的人""你在搭每天自动产出结论的系统"——**把读者塑造成"系统建造者 / AI 协作者"**。

**用"你"重塑读者身份**，是他的传播术。

### B5. 中文场景化具名："Sarah 还记得 / Ravi 还记得 / 张三明天走"

不像歸藏直接用产品名，也不像卡兹克起绰号。小互**用中性虚构人名做场景化具象**：
- 公司大脑篇：Sarah、Ravi、销售副总、产品同事、张三、小李
- OpenClaw 篇："出差的人""跑客户的""做零售的""开会多的"

这些人不是真名，是模型——**把"抽象的痛点"映射到"具体的人脸"**。读者能想到自己公司里对应的那个人。

### B6. 关键术语必带原文 + 中译 + 解释三件套

英文术语三件套写法："transactive memory，翻译过来就是「团队里互相知道谁懂什么」的那种默契" / "metacognition，说人话就是「想自己有没有想清楚」"

这种"术语三连"在 7000 字深度解读里出现 5+ 次。**不省术语（专业感）+ 不省翻译（友好感）+ 不省解释（说服感）三件套**。这条工程量大但回报高——非英文圈读者能跟上，英文圈读者也认可专业度。

### B7. 数据 / 数字密集塞入

深度解读篇里出现的数据点：
- "今年 1 月拿了 500 万美元种子轮"
- "1991 年就有过一篇基础论文"
- "8 成的企业把数据问题列为扩展 agentic AI 的头号障碍"
- "覆盖 100 多个数据源"

实测复盘篇：
- "已抓取并写入 80 期历史数据"
- "20 条真实竞品数据"
- "覆盖率 >=90%"
- "TOP5 样本 100% 可追溯"

**数字给"翻译者"和"实测者"双重身份做证据**。这条跟"小互的内容很克制、不喂垃圾"那条业内评价相符——**数据密度撑信任**，不靠形容词撑。

### B8. 情绪温度低，但偶尔爆"哈哈哈😁 / 牛P / 兴奋了一整天"

实测复盘篇出现：
- "兴奋了一整天"（开篇）
- "哈哈哈😁"（独立成段）
- "出乎我的意料，非常的详细..."
- "非常的牛P..."

但深度解读篇情绪几乎为零。**情绪只在"自己跑出来的实测"里释放**，对别人的产品 / 论文做翻译时保持完全冷静。

跟卡兹克的"段段灌情绪"和歸藏"全篇平稳"都不同——**小互是"按内容类型切换情绪开关"**。

## 跟卡兹克 + 歸藏对照表

| 维度 | 卡兹克 | 歸藏 | 小互 |
|---|---|---|---|
| 段均字数 | 24 | 25-50 | 44-57 |
| 篇幅 | 2500-3300 | 1700-4600 | 5000-7000 |
| 第一人称密度 | 1/50 字 | 1/100-200 字 | **类型切换**：1/115 实测 / 1/700 深度 |
| 开场套路 | "昨天，X" + 戏剧化 | "我现在 / 上周" + 直给 | 具体场景 → 引出海外人物 / 自己产品 |
| 收尾节奏 | 长 → 短 → 口号 | 平静总结 + 链接 + 转发 | "留个问题给你"反向钩子 |
| 章节切分 | 看类型决定 | 几乎都轻分（疑问句标题）| 数字编号"01-09"，标题就是观点 |
| 配图密度 | 1/100 字 | 1/270 字 | **类型切换**：1/165 实测 / 1/875 深度 |
| 标志性句式 | 短句情绪爆点（"恐怖如斯"）| 设计审美判断（"约束越严"）| 大白话翻译（"翻成一句人话"）|
| 引用密度 | 几乎无 | 几乎无 | 高（论文 / 报告 / 人名 / 产品 / 时间）|
| 物理化动词 | 多 | 极少 | 极少 |
| 口语爆点 | 多（YYDS / 狗东西）| 几乎无 | **类型切换**：实测有（牛P）/ 深度无 |
| 风格定位 | 老哥跟你聊 | 设计师 + 开发者 | 翻译者 + 实测者 |

**结论**：三人覆盖三条不同路线：
- **卡兹克 = 情绪派**：靠节奏 + 情绪锚点 + 配图密度撑读完率，破圈能力最强
- **歸藏 = 结构派**：靠并列 + 列表 + 中英混排撑专业感，垂直圈层最稳
- **小互 = 翻译派**：靠"长块 + 断点 + 翻译"撑信任，**给中文读者搬运英文世界的桥梁**

## 定位标签

```
小互 = 翻译者 × 实测者 × 双模写作
```

三重身份叠加：
- **翻译者**：英文论文 / RFS / 海外创业公司原文 → 中文长文。每个术语三件套（原文 + 中译 + 解释）
- **实测者**：自有 Agent + 多平台测试 → 4 个实验 + 落地结果。带验收标准
- **双模写作**：A 类深度解读和 B 类实测复盘是两套不同模板，配图密度 / 第一人称 / 情绪强度都不同

**用户记忆里"速报体 + 标题党"的小互可能是 X / 推特上的 imxhy（500-1500 字短播报），这跟公众号「小互AI」的写法不同**。公众号上他走的是长文翻译路线，篇幅 5000-7000 字。如果要复刻"短平快速报"，需要参考他在 X 上的内容，本指纹覆盖不到这一面。

## 什么场景调他

写下面这些类型的稿子，可以参考小互的指纹：

| 场景 | 该抄哪几条 |
|---|---|
| 给中文读者翻译英文论文 / 海外报告 | A2 数字章节 + B1 翻译三连 + B6 术语三件套 + A5 来源全标 |
| 自有产品多场景实测复盘 | A3 实验 1-N + 落地结果 + A4 反向钩子 + B7 数字密集 |
| 拆解海外创业公司商业模式 | A5 引用 + B2 大白话压判断 + B5 中文具名场景化 |
| 长文（5000+ 字）撑得住 | A6 块 + 断点节奏 + B4 你公司 / 你的 Agent 重塑读者 |

**反过来**——下面这些场景**不要**抄小互：
- 想写 1500 字以内短稿：他的"长块 + 断点"节奏需要篇幅展开，短稿撑不开
- 想冲个性 / 鲜明立场：他的翻译者口径是"代他人发声"，立场感弱于歸藏和卡兹克
- 写完全不引用外部材料的纯个人随笔：他的引用密度是支柱，没引用就塌

## 改写示例

取我们之前的 12 components 文章原段：

> 原文：
> GitHub 上 shareAI-lab 的 `learn-claude-code` 把这件事掰得很清楚——12 节课，12 个递进的机制，每章在前一章基础上叠加一个东西。我读完 12 章源码（每章 100-800 行 Python，加起来 4000 多行），读完反而发现，外界普遍流传的那张"12 组件平铺图"是错的。

按小互"深度解读"风格重写：

> 一个反常识的事实：你在网上看到的所有 Claude Code 架构图，可能都是错的。
>
> shareAI-lab 上周开源了一份叫 learn-claude-code 的代码库（github.com/shareAI-lab/learn-claude-code）。这个仓库今年 3 月发布以来已经积累了 1.2k star，作者把 Claude Code 的内部拆成 12 节课，每节 100-800 行 Python，加起来 4000 多行。
>
> 这是目前我能找到的最逆向工程到底的版本。
>
> 翻成一句人话：作者把 Claude Code 内部当成洋葱拆，每剥一层就单独写成一节课，让你看清"这一层做了什么、为什么需要它、它依赖前面哪一层"。
>
> 读完之后你会发现一件被普遍误解的事：
>
> 外界流传最广的那张"12 组件平铺图"——把 12 个组件画成 12 个并列方块——结构上是错的。
>
> 真实的结构不是平铺，是叠加。
>
> 12 层依次往上盖，每一层在前一层基础上加一个机制。

差异点（跟卡兹克 / 歸藏对比）：
- 开场上"反常识的事实"（A2 章节标题就是观点）
- "今年 3 月发布以来已经积累了 1.2k star" 数字密集（B7）
- "翻成一句人话"招牌动作（B1）
- "读完之后你会发现" 用第二人称重塑读者（B4）
- "12 层依次往上盖" 是块叙事中间的断点（A6）

注意：示例里的 1.2k star 是为示范结构编的，实际写时小互会去查证。**他的"翻译者"身份要求来源真实**。

## 怎么用这份指纹

跟另外两份对比：

| 内容类型 | 优先调谁 |
|---|---|
| 海外论文 / 报告 / 创业公司翻译解读 | **小互**（翻译者口径无人能替）|
| 自有 Agent / 工具的多场景实测复盘 | **小互**（实验 + 落地结果模板最规范）|
| 工具评测、产品发布、技术教程 | 优先歸藏（A2 + A3 + B3 + B6）|
| 设计 / 审美 / 方法论自述 | 优先歸藏（A5 + B2 + B7）|
| 行业反思 / 立场表达 / 黑暗森林型 | 优先卡兹克 |
| 5000+ 字长文 | 抄小互的 A6 块 + 断点节奏 |

**三人混用建议**：
- 长文翻译稿：主体抄小互，开场局部抄卡兹克"昨天 X" 提速
- 工具实测稿：主体抄歸藏的并列结构，结尾抄小互"留个问题给你"反向钩子
- 设计 / 审美深度稿：主体抄歸藏的判断句，开场抄小互的"反常识事实"

## 反例 / 警惕

- **"翻成一句人话"是双刃剑**：用得对是亲和，用太多次会显得不专业（"为什么你不一开始就说人话"）。控制在 7000 字里 3-5 次为宜。
- **"不是 X 而是 Y"句式命中 critique 硬规则**：小互能撑住是因为后接具体例子。如果只用句式不带例子，会立刻像 AI 写的。**抄之前必须确认每个"而是"句后面都跟具体场景或人**。
- **数字密集 ≠ 编造数字**：他的数字密度是支柱，但每个数字都对应真实来源（论文 / 报告 / X 推文）。如果数字编造或来源虚构，整篇信任坍塌。**这是引用派最大的红线**。
- **块状段落不等于水文**：他的"长块 + 断点"是有节奏的，每个长块都在推进一个论证步骤，每个断点都在做转折或定锚。如果只学"长段"不学"节奏断点"，会变成水文。
- **5000+ 字篇幅是门槛**：小互的所有结构（数字章节 / 术语三连 / 长块断点 / 多场景实测）都需要篇幅展开。如果目标稿件是 1500 字以内，**强行抄小互会显得拥挤、不透气**。
