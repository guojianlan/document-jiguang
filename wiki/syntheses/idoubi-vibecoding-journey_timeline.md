## 4 次范式跃迁的内在逻辑

| 跃迁 | 触发器（模型能力）| idoubi 的策略转向 |
|---|---|---|
| GPTs → 套壳 | GPT-4 API 开放 | 从"教别人 GPTs"转到"做应用层套壳" |
| 套壳 → MCP | Anthropic MCP 发布（2024-11）+ Manus 爆火（2025-03）| 从"做应用层"转到"做平台基础设施"——mcp.so + 出书 |
| MCP → Vibe Coding | Claude Code Max 顶配能力 + Claude Agent SDK | 从"写代码做产品"转到"指挥 agent 做产品" |
| Vibe Coding → ? | 待观察 | 2026-04 流露出"意义焦虑"——**下一次转向可能是更窄、更深，而不是更宽** |

## 稳定的方法论锚点

无论范式怎么变，idoubi 始终在用同一套：

1. **5 步 SOP**：Ideation → Development → Launch → Growth → Monetization
2. **标准技术栈**：NextJS / Supabase / Stripe / Cloudflare
3. **3 大增长**：ProductHunt + 程序化 SEO + AI Wrapper
4. **影响力变现**：把过去积累的关注度一次性兑现（出书 / SaaS 框架预售）

这套方法论的有效性在 3 年里被反复验证——但 2026 年开始**遭遇"护城河模糊"挑战**：当所有人都能用 AI 做产品时，什么是稀缺？idoubi 自己的答案是"流量 + 商业 + 知识 + 架构"——但还没找到具体抓手。
