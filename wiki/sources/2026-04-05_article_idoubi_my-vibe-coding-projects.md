# 我的 Vibe Coding 项目（最新）

## 一句话摘要

3 个月全自动 Vibe Coding 9 个产品矩阵——WorkAny / FastClaw / ChatClaw / CoRich / WeClaw / AnyClaw / open-agent-sdk 等组成 OpenClaw 完整生态闭环。

## 关键事实

- **WorkAny**：Open Agent SDK 替换 Claude Agent SDK，1.4k Stars
- **FastClaw**：Go 实现 OpenClaw 替代，秒级启动 / 内存 20MB（vs 140MB）/ 3000 行 / RPC 进程隔离 / 云原生多租户
- **ChatClaw**：OPC → Companies → Teams → Agents → Tasks 分层架构，解决 OpenClaw 丑界面
- **CoRich**：Agent 版 Slack（对标微信 ClawBot 封禁中国公司）
- **WeClaw**：微信 ClawBot 桥接，自动探测电脑 Agent，ACP/CLI/HTTP 三种连接
- **AnyClaw**：Agent 技能补充包，转换器 / 扩展坞 / 工具箱三层
- **open-agent-sdk**：从 Claude Code 泄露源码提取，TypeScript/Go/Rust/Python 四语言，2k Stars + 321k 访问

## 作者观点

- 全自动 Vibe Coding 让角色变成"项目经理 + 架构师 + 测试工程师"而非"程序员"
- AI 治好强迫症和代码洁癖
- **测试资源极度缺乏**——AI 快速迭代但人工验收跟不上
- **护城河模糊**——技术落地能力被 AI 抹平，流量 + 商业 + 知识 + 架构变成稀缺竞争力
- 时常迷茫："做这些产品的意义在哪里"，也许应该在一个方向持续做深

## 我的判断

- 9 个产品矩阵 = idoubi 完整 Agent 生态战略，但他自己最后一段流露出**深度焦虑**——"做这些产品的意义在哪里"，**这是 wiki 最该捕捉的反神话信号**
- "护城河模糊"判断与 [[2024-05-22_article_idoubi_ai-search-engine]] 的"支付率 0.03%"形成 2 年呼应链——2024 年是流量难变现，2026 年变成"全能力被抹平"
- "AI 治好强迫症和代码洁癖"——表面是优点，但和 idoubi 自己说的"测试跟不上"放一起，**暗示了质量退化的代价**
- open-agent-sdk 从 Claude Code 泄露源码提取——这是 [[Claude Code]] entity 应该补的"开源衍生品"段
- 与 [[2026-01-24_article_idoubi_vibe-coding-workany]] 形成"季度演进"对照：3 个月从 1 个产品（WorkAny）变成 9 个，**生产速度极快**，但作者本人开始反思"快不如精"
