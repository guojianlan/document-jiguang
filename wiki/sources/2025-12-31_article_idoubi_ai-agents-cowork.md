# 我的 AI 员工会后空翻

## 一句话摘要

6 个 coding agent 的四轮能力评测——Agents.md 网页竞赛 / Agent Skills 调用 / 联网搜索 / MCP 工具 + 总分总协作模式实战。

## 关键事实

- **Agents.md 竞赛**：Codex+GPT5.2 质量最高（5x 慢），Claude Code 综合最强，Gemini 速度快但 AI 味浓
- **Agent Skills**：Claude Code 渐进式加载最强；Codex/Crush 输出质量最好；Gemini 单文件无拆分
- **联网搜索**：Gemini > Codex > Claude Code > Crush
- **MCP 挂载**：5 个 Agent 都能调高德地图 MCP 完成驾车规划
- **总分总协作**：Claude Code 架构师 / Gemini 后端 / Codex 前端 / Crush 测试，全栈协作完成 todolist

## 作者观点

- Agent 评估需多维量化（速度 vs 质量权衡）
- Agents.md + Agent Skills + MCP 是完整赋能栈，三者互补
- "群体自治"模式（多 Agent 自治 + 协作 + 监督 + 执行）最有价值

## 我的判断

- 这篇是 wiki 现有 [[Claude Code]] 单 agent 视角的**强力补全**——加入了横向对比的批判性视角（Claude Code 不是处处第一）
- "Agents.md + Agent Skills + MCP 三者互补"印证了 wiki 已有 [[AI 工作流]] 概念里的"四条路线不冲突"判断
- 但要注意：idoubi 的横评是**单点任务**评测，不能直接推广到 long-running 复杂项目——典型的"评测有效性"问题
- 总分总模式 vs [[Superpowers]] 的"三级审查"模式：前者是空间维度的分工，后者是时间维度的串行——两种 multi-agent 编排正交
