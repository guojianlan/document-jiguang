# MCP is all you need

## 一句话摘要

idoubi 的 MCP 本质论——三阶段发展（发布 / 起势 / 爆火）+ 双边网络效应 + vs Function Calling 三维对比 + 5 大生态组成，得出"MCP 是 AI 时代 HTTP 协议"的判断。

## 关键事实

- MCP 三阶段：2024-11 发布 → 2025-01 Cursor / Perplexity 支持 → 2025-03 Manus 爆火
- vs Function Calling 三维差异：协议标准 / 功能丰富（资源 + 提示词 + 工具）/ 一次实现处处可用
- 5 大生态：服务器 → MCP App Store → 服务路由（OpenRouter 类比）→ chatbot/agent → CLI 工具
- 4 大落地场景：企业 AI 工作台 / 个人 AI 助手 / Agent / MCP 服务路由

## 作者观点

- "MCP 一定程度也可理解为 AI 时代的 HTTP 协议"
- "MCP 是秦始皇统一六国的现代版"——统一接口标准加速行业
- "MCP is all you need"——是 AI 应用必备基础设施

## 我的判断

- 标题《MCP is all you need》明显是夸张话术——任何说"X is all you need"的论述都要打折听
- 但"鸡蛋悖论 → 明星效应 → 双边网络效应"的双边市场模型确实是 MCP 起势的最佳解释
- vs Function Calling 三维对比是这篇最实在的部分，可作为 [[MCP]] entity 的标准 KO 论据
- "AI 时代 HTTP 协议"的比喻**比 LSP 比喻弱**——HTTP 是端到端通信协议，MCP 是工具调用协议，类比对象不对
