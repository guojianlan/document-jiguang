# 我写了一款 AI-Native 产品：知了 zKnown

## 一句话摘要

知了 zKnown 从文章摘要工具迭代为 AI-Native 个人知识助手——支持智能摘要、内容剪藏、向量语义检索，多端覆盖（微信 / Web / Mac / Win / Linux / iOS）。

## 关键事实

- 3 大功能：智能摘要、内容剪藏、向量语义检索
- 摘要输出：一句话、关键点、思维导图、标签
- 技术栈：Prompt Engineering + Embedding + 向量数据库 + 无头浏览器
- 获奖：极客公园 Hackathon 创意赛第三、阿里云创客松冠军

## 作者观点

- AI-Native 产品应从用户真实需求（信息降噪）出发，不是模型堆砌
- 开放 API 比单一产品更有生态价值

## 我的判断

- 这是 idoubi 早期"AI Native"概念的具体化——和后来的 [[MCP]] 时代相比，2023 年的 "AI-Native" 还偏向"AI 增强工具"，不是真正的 agent
- 与本仓库做的事情高度共振：知了的"摘要 + 剪藏 + 语义检索"几乎就是这个 wiki 的功能 spec
