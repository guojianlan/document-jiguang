# 内容处理工作流

这个仓库的核心流程现在有五步：

1. 收集来源
2. 拆解来源
3. 沉淀主题
4. 生成输出
5. 发布包装

## 1. 收集来源

来源可能包括：

- 文章链接
- 视频链接
- PDF
- Word / PPT / Excel / CSV
- 网页截图
- 推文串
- 播客
- 代码仓库

进入仓库时，先判断它属于哪一类，然后先放到 `raw/inbox`。

## 2. 拆解来源

拆解时优先提取这六类信息：

- 一句话摘要：它到底讲了什么
- 核心观点：作者想表达的主要结论
- 关键事实：数据、案例、定义、发布时间、产品变化
- 方法框架：步骤、模型、SOP、最佳实践
- 你的判断：哪些是事实，哪些是观点，哪些需要再验证
- 可复用素材：金句、提纲、图示思路、可转述表达

### 不同来源类型的处理重点

#### 文章 / 报告

- 看结构
- 看论证链
- 看是否有数据和案例支撑

#### 链接 / 新闻 / 产品页

- 看是否是最新信息
- 看是否需要补一手来源
- 看是否只适合做事实快照

#### 视频 / 访谈

- 看主线观点
- 看是否能拆成章节
- 看是否有值得保留的表达方式

#### PDF

- 先抽取文本，再做结构化整理
- 如果有图表，记录图表结论，不要只抄图表标题

#### 文档

- 先判断是资料型文档、演示型文档还是数据型文档
- 再提取结构、关键信息和可复用结论
- 如果是表格或演示文稿，不要只看标题，要补充页内实际信息

## 3. 沉淀主题

当多个来源都指向同一个问题时，不要只停留在来源笔记，应该建立主题笔记。

主题笔记至少回答三个问题：

- 这个主题是什么
- 目前有哪些稳定结论
- 还存在哪些待验证问题

推荐做法：

- 每个主题都建立一个中心笔记
- 把来源笔记通过双链挂过去
- 逐步把“来源摘要”升级为“你的观点”

## 4. 生成输出

输出时不要直接面对原始来源，而是优先从主题笔记生成内容。

输出类型可以包括：

- 内部分享文档
- 长文
- 短文
- 讲稿提纲
- 主题清单
- 研究总结

## 5. 发布包装

当内容已经进入“准备发布”的阶段，不要只停留在正文。

推荐继续生成：

- 发布版正文
- 发布建议
- 封面图
- 认知图 / 结构图
- 社交媒体切图文案
- SVG 图卡

## 我们以后协作时的默认动作

当你给我一个外部来源时，我默认按下面顺序处理：

1. 判断来源类型
2. 提炼摘要
3. 提取关键信息
4. 区分事实和观点
5. 提取可复用框架或表达
6. 标注适合归入的主题
7. 视情况继续生成主题笔记
8. 如果用户要做内容输出，再继续生成草稿
9. 如果用户要对外发布，再继续进入发布包装

## 建议的最小闭环

你可以先用这个最小动作启动仓库：

1. 丢给我 3 到 5 条你最近关注的 AI 内容
2. 我帮你转成结构化来源笔记
3. 我再把它们整理成 1 个主题笔记
4. 最后输出 1 篇可以分享的总结文档
