# 文章配图与图卡流程

这个流程用于把已经成型的文章内容，继续拆成可传播的视觉资产。

## 目标

不是只做“好看图片”，而是做：

- 能服务正文表达的封面图
- 能快速建立认知的结构图
- 能独立传播的图卡
- 可继续复用和修改的 SVG 源文件

## 标准流程

### 1. 先从正文里提炼可视化对象

优先找：

- 一句话总论
- 一条方法链路
- 1 到 3 个高传播句子

### 2. 判断图像类型

按内容性质选择：

- 总论句：封面图
- 框架 / 路径：认知图、结构图
- 传播句：社交图卡

### 3. 先生成 SVG 源文件

SVG 作为长期可编辑源文件，默认落在：

- `raw/attachments`

如果是文字密度高、容易超框的结构图或图卡，首次生成就优先改用：

- 结构化数据
- `Satori` 模板
- 再生成 SVG

说明见：

- [Satori视觉资产说明.md](/Users/apple/Desktop/project/document/99_System/Satori视觉资产说明.md)

### 4. 必做渲染验证

生成后必须至少跑一次：

```bash
bash scripts/validate_svg_asset.sh "raw/attachments/xxx.svg" --png "raw/attachments/xxx.preview.png"
```

对于文字型 SVG，建议再显式跑一次：

```bash
node scripts/check_svg_layout.js "raw/attachments/xxx.svg"
```

检查：

- 是否裁切
- 是否错位
- 是否行长过长
- 是否层级不清
- 是否有明显文字重叠

### 5. 需要分发时再导出 PNG

如果目标平台需要 PNG，再从已验证的 SVG 导出。

## 适用场景

- 公众号文章配图
- 小红书图文
- 社群传播图卡
- 系列内容视觉母版
